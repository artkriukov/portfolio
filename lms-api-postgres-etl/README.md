# ETL-пайплайн: API → PostgreSQL
Скрипт забирает из API историю попыток студентов по учебным задачам, чистит данные и сохраняет их в [Supabase](https://supabase.com/).

## Проблема

Данные об отправках студентами решений задач по программированию доступны только через внешний API. Часть полей приходит в неструктурированном виде, записи могут быть неполными. Без очистки и загрузки в собственную БД их нельзя использовать для аналитики.

## Цель проекта

**Разработать ETL-пайплайн, который:**

- получает из API данные об отправках решений учебных задач по программированию
- приводит их к структурированному и валидному виду
- загружает результат в PostgreSQL
- логирует этапы работы и ошибки в файл с именем по текущей дате и хранит логи только за последние 3 дня
- считает дневные агрегаты и отправляет их в Google Sheets
- отправляет email-уведомление о результате запуска
